## 修复英文小说搬运网站本地预览按钮失效问题

### 问题分析
用户反映在 Edge 本地 file:// 预览时，多个按钮点击没有反应。

### 根因
1. **`initReadingSettings()` null 访问危险**：函数在 null 检查（`if (!settingsBtn...)`）之前就直接赋值 `fontSizeSlider.value = settings.fontSize`，当元素不存在时抛 TypeError，中断整个初始化链。
2. **`initChapterNavigation()` 缺少 null guard**：`prevBtn.addEventListener(...)` 在 `prevBtn` 可能为 null 时直接调用，抛 TypeError。
3. **广告 push 脚本未加 try-catch**：各页面内联的 `adsbygoogle.push({})` 在本地 file:// 环境下可能因 AdSense 脚本异常加载而出错，影响页面初始化。
4. **`addToFavorites` 按钮无事件绑定**：novel.html 中收藏按钮点击后无任何反应（功能缺失）。
5. **Genre 过滤器大小写不匹配**：novels.html 过滤器选项值为小写（`fantasy`），但数据里 genre 是首字母大写（`Fantasy`），导致过滤始终无结果。

### 修复内容
1. `assets/js/app.js` - 修复 `initReadingSettings()` null 安全顺序
2. `assets/js/app.js` - 修复 `initChapterNavigation()` 添加 null guard
3. `assets/js/app.js` - 实现 `addToFavorites` 按钮收藏/取消收藏功能（使用 localStorage）
4. `assets/js/app.js` - 修复 genre 过滤器大小写匹配（`.toLowerCase()`）
5. `assets/js/app.js` - 整个 DOMContentLoaded 初始化用 try-catch 包裹，防止单个错误阻断其他按钮初始化
6. `index.html`, `novels.html`, `novel.html`, `read.html` - 所有广告 push 代码加 try-catch

## 修复 Edge "此页面没有响应" 卡死问题

### 问题
用户反馈用 Edge 打开 index.html 时弹出"此页面没有响应"对话框，整个页面卡死无法操作。

### 根因
**外部资源阻塞主线程**：页面同时加载 4 个外部资源（Google Fonts + Font Awesome CDN + Google Analytics + Google AdSense），在 `file://` 本地协议下 Edge 浏览器会同步等待这些跨域请求，导致主线程长时间阻塞，触发"无响应"警告。

### 修复方案（5个 HTML 文件全部修改）
1. **Google Fonts / Font Awesome CSS**：将 `<link rel="stylesheet">` 改为 `<link rel="preload" as="style" onload="...this.rel='stylesheet'">` 异步加载模式（非渲染阻塞）
2. **Google Analytics / AdSense JS 脚本**：添加 `file://` 协议检测，本地模式下完全跳过加载：
   ```js
   var isLocal = window.location.protocol === 'file:';
   if (!isLocal) { document.write('...external script...'); }
   ```
3. **所有广告 push 脚本**：增加协议判断，本地模式下不执行 `adsbygoogle.push({})`

### 涉及文件
- `index.html` - 头部3处+广告3处
- `novels.html` - 头部1处+广告2处
- `novel.html` - 头部1处+广告3处
- `read.html` - 头部1处+广告4处
- `about.html` - 头部1处

## 修复 GitHub Pages 图片不显示 + 章节数据不完整

### 问题
用户推送 GitHub 后，线上页面 (novelhub.ccwu.cc) 出现两个问题：
1. **封面图显示占位符** "The image is generating... Please refresh page to preview"
2. **章节数量过少**（每部仅 3-10 章），与本地完整数据不一致

### 根因分析
1. **图片问题**：所有数据文件（data-lite.js / data.js / novels.json）的 `cover` 字段均使用 `https://picsum.photos/seed/xxx/300/400`，该随机图片服务在 GitHub Pages 上无法正常加载，显示为占位符
2. **数据问题**：index.html 仅引入 `data-lite.js`（17KB 精简版），而包含完整章节内容+更多章节的 `assets/data/novels.json`（279KB）和 71 个独立章节 JSON 文件未被加载

### 修复内容 (`assets/js/app.js`)
1. **新增 `getCoverUrl(novel)` 函数**：检测 picsum.photos 等 URL 时自动替换为 SVG data-URI 渐变占位图（根据小说 ID 确定性生成颜色和首字母），确保任何环境下图片都能正常显示
2. **改造 `loadData()` 为 async 函数**：
   - 先加载内联 data-lite.js 作为兜底
   - 再通过 `fetch('assets/data/novels.json')` 加载完整数据（含章节正文）
   - GitHub Pages 上 fetch 成功后用完整数据覆盖精简数据
3. **所有图片引用替换**：`createNovelCard()`、`renderLatestChapters()`、`loadNovelDetail()` 中的 `<img src>` 全部改用 `getCoverUrl()`
4. **DOMContentLoaded 改为 async** 以支持 await loadData()
